Auth reasoning

Why BankID was dropped for MVP

Requires a formal broker agreement, onboarding time, per-authentication cost, and compliance review. In v1 users are just uploading their own files — no bank connection, no verified identity needed.

What we use instead

Google OAuth (primary) + magic link email (fallback). Configured in Supabase Auth dashboard quickly. Library: @supabase/ssr. No passwords, no Vipps, no broker agreements.

Categorization system

The core of the product. Rule-based in v1. No ML. Target: ~60–70% auto-categorized. Prioritize correctness and trust over automation.

💳 Debt Payments
💼 Business Expenses
❓ Uncategorized
Rule

Uncategorized is always visible — never hidden. Never force a low-confidence guess. Never show a confidence score to the user.

3-layer matching pipeline

1

Exact merchant match

Seeded dictionary of Norwegian merchants. Normalize first, then match.

2

Keyword / pattern rules

Versioned JSON rule config. Applied only if exact merchant match misses.

3

Fallback → Uncategorized

Everything unmatched. Never silently assigned.

Correction UX

Tap badge → category picker → select → instant save. Undo snackbar 3 seconds. No confirm dialogs. Stores category_source = 'user' and corrected_at.
